PROJECT OVERVIEW
- Goal: Build a Spotify-style song recommendation engine that suggests tracks similar to a chosen song using audio features and metadata.
- Current state:
  - A single notebook (`Notebooks/dataProcessing.ipynb`) loads `dataset/dataset1.csv`, drops an `Unnamed: 0` column, removes nulls, uppercases column names, explodes multiple artists into separate rows, converts `EXPLICIT` to int, and creates `DURATION_MIN`.
  - A preliminary feature list is defined (danceability, energy, loudness, speechiness, acousticness, instrumentalness, liveness, valence, tempo), but no model training or recommendation logic is implemented.
  - `dataset/cleaned_dataset.csv` and `dataset/dataset2.csv` exist; they appear to be alternative/derived datasets but are not used in code.
  - No package/dependency file or application layer is present.

DATA PIPELINE
- Current handling:
  - Loads `dataset/dataset1.csv`.
  - Drops `Unnamed: 0`, removes nulls.
  - Standardizes column names to uppercase.
  - Splits multiple `ARTISTS` by `;` and explodes rows.
  - Converts `EXPLICIT` to 0/1.
  - Computes `DURATION_MIN` from `DURATION_MS` (note: currently divides by 6000, which should be 60000 for ms→min).
- Required preprocessing steps:

  - Handle duplicates (same track_id across multiple artists).
  - Normalize/standardize numeric features for similarity.
  - Remove or impute remaining missing values if any.
  - Validate/clean outliers (e.g., loudness, tempo).
- Suggestions for improvement:
  - Build a reproducible pipeline function/script (not just notebook).
  - Save a cleaned feature matrix and metadata table for fast lookup.
  - Add train/validation splits for evaluation, even for recommender metrics.

RECOMMENDATION SYSTEM DESIGN
- Content-Based Filtering:
  - Use track-level audio features (danceability, energy, etc.) to compute similarity.
  - Best fit for current data because `dataset1.csv` includes rich audio features and no explicit user interaction data.
- Collaborative Filtering:
  - Requires user–item interactions or implicit feedback (not currently in the repo).
  - Not applicable unless new user behavior data is added.
- Hybrid Approach:
  - Combine content similarity with popularity or user preferences if interactions are later collected.
- Best fit now:
  - Content-based filtering is the most practical and immediate approach using the existing dataset.

MODEL DETAILS
- Suggested algorithms:
  - Cosine similarity on standardized feature vectors.
  - k-NN (e.g., `NearestNeighbors` from scikit-learn).
  - Optional: matrix factorization only if user–item data becomes available.
- Input features:
  - `DANCEABILITY`, `ENERGY`, `LOUDNESS`, `SPEECHINESS`,
    `ACOUSTICNESS`, `INSTRUMENTALNESS`, `LIVENESS`,
    `VALENCE`, `TEMPO`, (optionally `DURATION_MIN`).
- Recommendation flow:
  - Given a track ID, fetch its feature vector.
  - Compute similarities to all tracks.
  - Return top-N with metadata (track_name, artists, album, genre).
